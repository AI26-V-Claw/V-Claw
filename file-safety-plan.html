File Safety Gate

Spec ngắn cho lớp kiểm tra file trước khi agent đọc sâu, parse, gửi model, upload ra ngoài hoặc đưa vào tool.

Mục tiêu

  • Không tin filename, extension, MIME khai báo hoặc metadata từ nguồn ngoài.
  • Chặn file nguy hiểm trước khi parse, OCR, đọc vào LLM context hoặc upload sang Gmail/Drive/tool khác.
  • Mọi quyết định phải có lý do ngắn cho user và dữ liệu đủ chi tiết cho audit log.

Cách Triển Khai

  • Ưu tiên thêm module nhỏ, API rõ ràng, test riêng; hạn chế sửa rộng các file đang có sẵn.
  • Nếu một điểm tích hợp bắt buộc phải sửa file hiện có thì sửa trực tiếp, scoped nhỏ, không refactor lan sang phần không liên quan.
  • Giữ contract ổn định để các team khác chỉ cần gọi gate chung thay vì tự viết logic scan riêng.

Điểm Gắn Bắt Buộc

Luồng Gate cần đặt
filesystem.readFile Scan trước os.ReadFile; không đọc toàn bộ file lớn/nhị phân rồi mới truncate.
gmail.downloadAttachments Tải vào quarantine, scan bytes thật, sau đó mới promote vào workspace.
Telegram attachments Scan file tải về trước khi thêm vào Attachment paths cho agent.
drive.saveFile Scan nội dung Drive tải về trước khi ghi ra workspace.
drive.uploadFile và Gmail draft attachments Scan local file trước khi upload ra ngoài.
Archive Scan container trước khi giải nén và scan từng entry trước khi dùng.

Core Checks

  • Detect type bằng magic bytes/MIME sniffing; so với extension chỉ để cảnh báo hoặc chặn.
  • Giới hạn size: rỗng, quá nhỏ bất thường, quá lớn, hoặc vượt giới hạn parse/upload.
  • Chặn executable/script/shortcut: .exe, .dll, .scr, .bat, .cmd, .ps1, .lnk, .url.
  • Detect active content: Office macro/OLE, PDF JavaScript/Launch/OpenAction/embedded files, HTML/SVG script hoặc event handler.
  • Archive guard: zip slip, symlink, nested archive, số file quá lớn, tổng size giải nén quá lớn.
  • Parse nhẹ các type quan trọng để phát hiện file corrupt trước khi xử lý sâu.
  • Scan prompt injection sau khi extract text; luôn bọc nội dung file là untrusted content.

Decision Model

Decision Dùng khi
allow Type được hỗ trợ, size hợp lệ, extension khớp hoặc lệch không đáng kể, parse nhẹ thành công.
requires_approval Unknown type, thiếu extension, mismatch nhẹ, hoặc prompt injection nghi ngờ nhưng chưa đủ để chặn.
block Executable/script giả tài liệu, active content nguy hiểm, archive nguy hiểm, file corrupt, quá lớn, type bị cấm.

ask_user nếu dùng ở UI phải map về requires_approval trong backend/audit.

Output Contract

  • decision, flags, reason_user, reason_audit.
  • filename, extension, claimed_mime, detected_type, size_bytes, sha256.
  • origin, source_tool, scanner_version, policy_version.
  • Flags chính: external_origin, type_match, type_mismatch, dangerous_type, unsupported_type, too_large, too_small, corrupt_or_unreadable, archive_risk, active_content, prompt_injection_suspected.

Code Cleanup Đi Kèm

  • Sửa PathGuard.Resolve: không dùng prefix string để check root; dùng filepath.Rel hoặc boundary separator.
  • Thêm quarantine/promote API để tool không ghi file chưa scan thẳng vào workspace.
  • Thêm audit event hoặc metadata cho file safety decision; không nhét chi tiết scan vào text tự do.

Test Bắt Buộc

  • Valid PDF/image/text/docx/csv được allow.
  • EXE đổi tên thành PDF, HTML đổi tên thành PDF, script/shortcut từ nguồn ngoài bị block.
  • File rỗng, file quá lớn, PDF/docx/zip corrupt bị block hoặc requires_approval theo policy.
  • Zip traversal, symlink, nested archive, decompressed-size vượt giới hạn bị block.
  • Prompt injection trong text/PDF/HTML/docx bị flag và không được đưa vào model như trusted instruction.
